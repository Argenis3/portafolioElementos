Arduino básico entradas digitales

Argenis Emanuel Aragón López
Ingeniería en sistemas computacionales
Fecha: 09 05, 2024
Arduino Electrónica

Resumen

En esta práctica de Arduino básico se desarrollaron siete ejercicios que exploraron el manejo de entradas digitales y el uso de botones para controlar LEDs mediante condiciones lógicas. Comenzando con un botón simple para encender y apagar un LED, progresamos a utilizar dos botones para ampliar el control. Posteriormente, se introdujeron condicionales que determinan acciones basadas en el estado de los botones, aplicando tanto operadores lógicos "OR" como "AND" para crear circuitos más complejos.

Introducción

En esta práctica, se abordará la programación de un sistema basado en microcontroladores para la adquisición y procesamiento de datos. Los microcontroladores son dispositivos clave en el desarrollo de sistemas embebidos, permitiendo controlar múltiples dispositivos.

La finalidad de esta práctica es lograr entender el uso de de código para la programación de microcontroladores, así como en la comprensión de los elementos básicos de hardware involucrados en el proceso.

Materiales

  • Arduino Uno
  • 4 LEDs
  • 5 resistencias de 220Ω
  • 2 botones pulsadores
  • Cables de conexión
  • Protoboard
  • Computadora con software Arduino IDE
  • Cable USB tipo B para conectar Arduino

Desarrollo

Durante la práctica se realizaron 7 ejercicios diferentes utilizando un Arduino Uno, dos botones, cuatro LEDs, y resistencias. Cada ejercicio se centró en aprender cómo manejar entradas digitales con botones, y cómo controlar LEDs con diferentes tipos de condicionales y lógica básica de programación.

A continuación se describen los ejercicios desarrollados:

Ejercicio 1: Entrada Digital con un Botón

En este primer ejercicio, se conectó un botón al Arduino para aprender a leer su estado (presionado o no). El LED rojo se encendía al presionar el botón y se apagaba al soltarlo.

Ejercicio 2: Entrada Digital con 2 Botones

En este ejercicio, se utilizaron dos botones. Al presionar el primer botón, se encendía el LED rojo, mientras que el segundo botón encendía el LED verde.

Ejercicio 3: Entrada Digital con Condicionales y un Botón

En este ejercicio, se aplicó una condición para cambiar el comportamiento del botón. Si el botón estaba presionado, se encendía el LED amarillo, de lo contrario permanecía apagado.

Ejercicio 4: Entrada Digital con Condicionales y 2 Botones

Se implementaron condicionales para controlar dos LEDs con dos botones. Si ambos botones estaban presionados, los LEDs (rojo y verde) se encendían simultáneamente.

Ejercicio 5: Controlador de LEDs

Este ejercicio se centró en el control simultáneo de cuatro LEDs (rojo, verde, amarillo y azul). Los LEDs se encendían y apagaban secuencialmente dependiendo de la interacción con los botones.

Ejercicio 6: Condicional OR con Botones

En este ejercicio, se aplicó una condición lógica OR entre los dos botones. Si cualquiera de los dos botones estaba presionado, se encendía el LED azul.

Ejercicio 7: Condicional AND con Botones

Finalmente, se implementó una condición lógica AND. Ambos botones debían estar presionados simultáneamente para encender el LED amarillo.

En cada uno de los ejercicios, se hizo uso de resistencias de 220Ω para proteger los LEDs y evitar que se dañaran por un exceso de corriente.

Simulación

Utilizando la plataforma Tinkercad, se simularon los ejercicios desarrollados para verificar el correcto funcionamiento del circuito y el código antes de implementarlos físicamente. A continuación, se describe cada una de las simulaciones realizadas:

1. Simulación de Entrada Digital con un Botón

En esta simulación, se conectó un botón al Arduino y un LED a un pin digital. El código fue programado para encender el LED cuando el botón fuera presionado (estado HIGH). Tinkercad permitió verificar la correcta interacción entre el botón y el LED en el entorno virtual, simulando el circuito antes de montarlo físicamente.

2. Simulación de Entrada Digital con 2 Botones

Se simuló el funcionamiento de dos botones conectados al Arduino para controlar dos LEDs. Al presionar el primer botón, el LED rojo se encendía, mientras que al presionar el segundo botón, el LED verde se activaba. Esta simulación confirmó que el código podía manejar correctamente varias entradas digitales.

3. Simulación de Entrada Digital con Condicionales y un Botón

En este ejercicio, se aplicaron condicionales para controlar el comportamiento del LED amarillo. La simulación en Tinkercad permitió verificar que el LED respondía de acuerdo con la condición programada: encenderse solo cuando el botón estaba presionado.

4. Simulación de Entrada Digital con Condicionales y 2 Botones

Para este ejercicio, se utilizó la lógica condicional para controlar dos LEDs. Si ambos botones estaban presionados, los LEDs rojo y verde se encendían simultáneamente. Esta simulación ayudó a probar y ajustar el código que manejaba varias entradas y salidas digitales.

5. Simulación de Controlador de LEDs

En esta simulación, se conectaron 4 LEDs (rojo, verde, amarillo y azul) a diferentes pines digitales del Arduino. El código permitió alternar el encendido y apagado de los LEDs en un patrón secuencial. Tinkercad facilitó la verificación de la lógica de control de múltiples salidas digitales en el circuito.

6. Simulación de Condicional OR con Botones

En este ejercicio, se utilizó una lógica OR para controlar el LED azul. Si cualquiera de los dos botones estaba presionado, el LED azul se encendía. La simulación en Tinkercad ayudó a confirmar que el código funcionaba correctamente con la lógica OR aplicada a las entradas digitales.

7. Simulación de Condicional AND con Botones

Se simuló la lógica AND, donde ambos botones debían estar presionados simultáneamente para encender el LED amarillo. La simulación permitió verificar que el LED solo respondía cuando ambas entradas digitales estaban en estado HIGH.
