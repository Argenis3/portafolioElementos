Práctica 5: Attiny85 Outputs

Argenis Emanuel Aragón López
Ingeniería en sistemas computacionales
Fecha: 17 10, 2024
Arduino Electrónica

Resumen

En esta práctica se exploraron diversas salidas digitales del ATtiny85, utilizando diferentes dispositivos como LEDs, motores, buzzer, servomotores y tiras de Neopixels. Se desarrollaron 17 ejercicios que integran pulsadores para control manual y PWM para variaciones precisas de intensidad o velocidad. Además, se comprobó el control simultáneo de múltiples salidas mediante configuraciones avanzadas, maximizando el uso del microcontrolador.

Introducción

El ATtiny85 es un microcontrolador compacto y económico que permite realizar múltiples tareas de control digital y analógico. Esta práctica se centró en explorar las salidas del ATtiny85, como LED RGB, buzzer, motores DC, servomotores y tiras Neopixels. Además, se utilizaron pulsadores para interactuar con los dispositivos y PWM para controlar la intensidad de LEDs y velocidad de motores. El propósito fue comprender cómo se manejan diferentes tipos de salidas desde un microcontrolador sencillo pero versátil, como el ATtiny85, en situaciones cotidianas de control de dispositivos.

Materiales

  • 1 Arduino Uno
  • 1 Attiny85
  • 3 LEDs(RGB y comunes)
  • Resistencias de 220
  • Cables de conexión
  • Fuente de alimentación externa
  • Botones de pulsación
  • Buzzers
  • Tira de 5 leds neopixel

Desarrollo

Cada ejercicio fue previamente simulado en Tinkercad, permitiendo verificar las conexiones y el funcionamiento del código antes de realizar el montaje físico. La simulación facilitó la detección de errores en las conexiones y ajustes en los valores PWM. Los ejercicios que involucraban control por botones se probaron virtualmente para asegurar que los estados HIGH y LOW respondieran correctamente.

Simulación

  • 1. ATtiny85 LED con 2 Botones: Dos pulsadores permiten controlar el encendido y apagado de un LED.
  • 2. ATtiny85 LED Blink con 2 Botones:
  • Se configura un parpadeo controlado por botones, alternando entre dos velocidades de blinking.
  • 3. ATtiny85 Buzzer con 2 Botones
  • Un buzzer emite un tono continuo controlado por dos botones.
  • 4. ATtiny85 LED RGB con 2 Botones
  • Un LED RGB cambia su color mediante la combinación de tres canales (rojo, verde y azul).
  • 5.ATtiny85 LED RGB Contador con 2 Botones
  • Los botones incrementan o decrementan un contador que selecciona colores del LED RGB.
  • 6. ATtiny85 Motor DC con 2 Botones
  • Un motor DC se enciende o apaga según la entrada de dos pulsadores.
  • 7. ATtiny85 LED PWM
  • Control de la intensidad de un LED mediante modulación por ancho de pulso (PWM).
  • 8. ATtiny85 LED PWM con 2 Botones
  • Dos botones controlan la intensidad del LED mediante PWM.
  • 9. ATtiny85 Buzzer PWM
  • El buzzer emite diferentes tonos mediante PWM.
  • 10. El buzzer emite diferentes tonos mediante PWM.
  • Dos botones ajustan el tono del buzzer mediante PWM.
  • 11. Dos botones ajustan el tono del buzzer mediante PWM.
  • Control de la velocidad del motor DC mediante PWM.
  • 12. ATtiny85 Servomotor
  • Control de la posición de un servomotor
  • 13. ATtiny85 Servomotor con 2 Botones
  • Dos botones controlan el ángulo del servomotor.
  • 14. ATtiny85 Neopixel
  • Control de un LED Neopixel para cambiar colores programáticamente.
  • 15. Control de un LED Neopixel para cambiar colores programáticamente
  • Dos botones cambian el color del LED Neopixel.
  • 16. ATtiny85 Tira Neopixel x5 - 2 Botones
  • Control de una tira de 5 LEDs Neopixel para cambiar patrones de color.
  • 17. ATtiny85 Tira Neopixel x5 - 2 Botones (Control Avanzado)
  • Implementación de patrones avanzados en la tira Neopixel.

    • Diseño

    El ATtiny85 se conectó a los dispositivos (LEDs, motores, buzzer, etc.) mediante sus pines de salida. Los pulsadores se conectaron con resistencias pull-down para evitar fluctuaciones en las señales de entrada. Los dispositivos que requieren mayor corriente, como el motor DC y las tiras Neopixel, se alimentaron con una fuente externa. La programación se realizó a través del Arduino Uno como ISP.

    Construcción

    Montaje del Circuito en Protoboard

    Se colocaron los componentes en la protoboard y se realizaron las conexiones siguiendo el esquema probado en Tinkercad. Los pulsadores se ubicaron con resistencias pull-down para evitar rebotes. El ATtiny85 se conectó al Arduino Uno para cargar el código utilizando el entorno Arduino IDE.

    Pruebas de ajustes

    Cada ejercicio se probó de manera individual, verificando el comportamiento esperado de los dispositivos. Se realizaron ajustes en el código para afinar los tiempos de parpadeo, el control PWM y los ángulos del servomotor. Las conexiones fueron revisadas para evitar falsos contactos o problemas de alimentación.

    Las conexiones se realizaron de acuerdo con el esquema previamente diseñado.

    Ajustes Finales

    Durante las pruebas finales, el ATtiny85 se alimentó desde el Arduino Uno a través del puerto USB. Sin embargo, también se probó con una fuente externa para asegurar la estabilidad al controlar múltiples LEDs simultáneamente.

    Resultados

    Control por Pulsadores: Se comprobó que los pulsadores funcionaban correctamente para activar y desactivar los dispositivos. Control PWM: Los ejercicios con PWM mostraron variaciones suaves en la intensidad de los LEDs y la velocidad del motor. Neopixels y Servomotores: Se logró un control preciso de los Neopixels y del servomotor en cada ejercicio. Simulación Física Exitosa: Todos los dispositivos respondieron de acuerdo con lo esperado, validando el diseño y la programación realizados.

    Conclusiones

    Esta práctica permitió explorar las capacidades del ATtiny85 para el control de salidas digitales y analógicas, integrando dispositivos como LEDs, motores y Neopixels. El uso de PWM fue clave para el control preciso de intensidad y velocidad. La simulación en Tinkercad facilitó la corrección de errores antes de la construcción física. El uso de pulsadores proporcionó una interacción intuitiva con los dispositivos, demostrando cómo se puede realizar un control eficiente utilizando un microcontrolador compacto. Esta experiencia sienta las bases para proyectos futuros más avanzados con el ATtiny85 y otros microcontroladores.
