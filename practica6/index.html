Práctica 6: Attiny85 inputs

Argenis Emanuel Aragón López
Ingeniería en sistemas computacionales
Fecha: 17 10, 2024
Arduino Electrónica

Resumen

En esta práctica se exploró el uso del ATtiny85 para trabajar con entradas digitales y analógicas mediante sensores. Se realizaron tres ejercicios en los que se controló el encendido y comportamiento de LEDs en función de las señales recibidas desde sensores digitales y analógicos. Las simulaciones se realizaron en Tinkercad, verificando la correcta integración entre sensores y actuadores en sistemas embebidos simples.

Introducción

El ATtiny85 es un microcontrolador pequeño pero poderoso, útil para proyectos que requieran control eficiente de entradas y salidas. En esta práctica, se estudió su capacidad para manejar señales digitales y analógicas provenientes de sensores, utilizando estos valores para controlar LEDs.

Materiales

  • 1 Arduino Uno
  • 1 Attiny85
  • 3 LEDs
  • 3 resistencias de 220
  • Cables de conexión
  • 1 sensor analógico
  • 1 sensor digital

Desarrollo

A continuación, se detalla el flujo y propósito de los 3 ejercicios realizados, destacando las funcionalidades clave exploradas con el ATtiny85 y el Arduino como programador.

  • Ejercicio 1 ATtiny85 Sensor Digital ON/OFF LED: Encender y apagar un LED utilizando un sensor digital.
  • Ejercicio 2 ATtiny85 Sensor Analógico ON/OFF LED un LED si el valor de un sensor analógico supera un umbral.
  • Ejercicio 3 ATtiny85 Sensor Analógico Delay LED: Modificar el tiempo de encendido y apagado del LED según la lectura del sensor analógico.

Simulación

  • 1. Sensor Digital ON/OFF LED:
  • En la simulación, al presionar el botón, el LED se encendió inmediatamente. Al soltarlo, el LED se apagó. Esto confirmó la respuesta rápida del ATtiny85 a señales digitales.
  • 2. Sensor Analógico ON/OFF LED:
  • Se ajustó el potenciómetro en la simulación, y cuando el valor superó 512, el LED se encendió. Esta simulación permitió validar el uso de entradas analógicas y la comparación de valores.
  • 3. Sensor Analógico Delay LED
  • Al cambiar el valor del potenciómetro, se observó cómo el LED parpadeaba más rápido o más lento. Esto demostró la capacidad del ATtiny85 para controlar tiempos de ejecución en función de entradas analógicas.

Diseño

El diseño de los circuitos se centró en conectar correctamente sensores y LEDs al ATtiny85. Se utilizaron resistencias pull-down en los sensores digitales y resistencias limitadoras para los LEDs. Además, se preparó el Arduino Uno como programador ISP para cargar el código en el ATtiny85 antes de ejecutar las simulaciones.

Construcción

Montaje del Circuito en Protoboard

Se siguió el esquema diseñado, conectando los componentes en la protoboard. El sensor digital se conectó con una resistencia pull-down para evitar lecturas incorrectas. Los potenciómetros se conectaron a pines analógicos.

Programación del attiny85

Se usó el Arduino Uno como programador ISP para cargar los códigos en el ATtiny85. Una vez programado, se realizaron pruebas físicas para asegurar el correcto funcionamiento.

Verificación del Funcionamiento

Después de completar el montaje, se verificaron las funciones de cada uno de los ejercicios, asegurando que: Se verificó que los LEDs respondieran adecuadamente a los sensores. Se realizaron ajustes menores en las conexiones para mejorar la estabilidad.

Resultados

Respuesta Digital: El LED respondió de inmediato a las señales del botón en el primer ejercicio. Control Analógico: El LED se encendió y apagó correctamente en función del potenciómetro en el segundo ejercicio. Control de Retardo: El último ejercicio mostró cómo se podía ajustar el tiempo de parpadeo del LED según la lectura del sensor analógico.

Conclusiones

Esta práctica demostró la versatilidad del ATtiny85 para manejar tanto entradas digitales como analógicas, aplicando los valores de los sensores para controlar LEDs de manera eficiente. Se logró validar la capacidad del microcontrolador para responder rápidamente a entradas digitales y ajustar salidas en función de lecturas analógicas. Estos ejercicios forman una base sólida para futuros proyectos con sistemas embebidos, demostrando cómo se pueden integrar sensores y actuadores en circuitos simples pero efectivos.
